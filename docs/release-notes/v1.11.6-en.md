## Overview

This release extends credential health inspection to Codex and xAI while aligning browser-local inspection in lightweight CPA Panel mode with Manager Server workflows. xAI uses billing-first checks by default and supports optional minimal real inference. Results distinguish billing, quota, authentication, model, and inference failures while providing responsive configuration, sanitized failure details, and clearer operational guidance.

## Highlights

### Features

- Support Codex and xAI Providers in credential health inspection, with consistent configuration, filters, quota windows, and result presentation across browser-local and Manager Server workflows (`web/monitoring`, `manager-server/codexinspection`).
- Check xAI billing status by default and optionally perform minimal non-streaming Responses inference through the inspected credential. Models, prompts, and failure classifications are handled per Provider (`web/monitoring`, `manager-server/codexinspection`).
- Distinguish billing, quota, authentication, model, and inference failures while preserving sanitized HTTP evidence and expandable failure details without exposing `auth_index` in result lists (`web/monitoring`).
- Make inspection configuration drawers, filters, status panels, and result cards responsive across desktop and mobile while aligning local and scheduled-server settings and action states (`web/monitoring`).

### Docs

- Reorganize the bilingual README around monitoring questions, screenshots, and panel selection, preserve the existing four screenshots, and add Runtime Model, backup, admin-key recovery, and legacy-migration links (`docs/readme`).

## Upgrade Notes

- Existing Codex-only inspection configuration remains compatible. No database migration or Compose, environment, route, or storage-protocol change is required.
- xAI real inference remains disabled by default. Enabling it sends a minimal non-streaming Responses request through each inspected credential.
- Lightweight CPA Panel mode supports browser-local inspection; scheduled inspection and historical results still require Manager Server.
