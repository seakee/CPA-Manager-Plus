# CPA Manager Plus v1.9.2

## Overview

This release focuses on AI Providers configuration and Codex quota observability: reset credits, Codex connectivity testing, and cross-provider disableCooling controls are now surfaced in the UI, while model discovery, plugin OAuth provider resolution, and monitoring filters receive targeted cleanup.

## Highlights

### Features

- Show available Codex rate-limit reset credits in quota state, with a usage-payload count fallback when the dedicated endpoint is unavailable (`web/quota`).
- Add disableCooling edit fields and detail-drawer quick toggles for Claude, Codex, Gemini, and OpenAI providers, including optimistic updates and rollback on failure (`web/ai-providers`).
- Add a Codex provider connectivity test in the edit drawer, with model selection and `/v1/responses` request validation for the current config (`web/ai-providers`).
- Improve model discovery by hiding already configured models, preventing duplicates, showing a compact inline "Added" badge, and adding select-visible / clear-selection actions (`web/ai-providers`).
- Honor explicit `oauth_provider` fields from plugin manifests so plugin OAuth keys do not collide with built-in provider ids (`web/oauth`).

### Fixes

- Hydrate OpenAI provider `disableCooling` from `/config` when `/openai-compatibility` omits the field, merging by normalized provider identity (`web/providers`).
- Show per-key OpenAI test failures through a shared status indicator, with content-sized error tooltips (`web/ai-providers`).

### Refactor

- Remove unused realtime monitoring filters for header error kind, header error code, and quota plan, reducing the filter bar from 9 dropdowns to 6 while preserving underlying event diagnostics (`web/monitoring`).

### Tests

- Add regression coverage for OpenAI key status tooltips, provider `disableCooling` hydration, Codex reset credits, provider requests, plugin OAuth provider parsing, and monitoring filter cleanup (`web`).

## Upgrade Notes

- No database migration or backend configuration change is required.
- Automation or bookmarks that still include the old monitoring URL parameters `headerErrorKind`, `headerErrorCode`, or `quotaPlan` will no longer affect filtering.
- Suggested screenshots for this release: AI Providers disableCooling controls, Codex connectivity test, model discovery "Added" badge, and Codex reset credits quota display.
