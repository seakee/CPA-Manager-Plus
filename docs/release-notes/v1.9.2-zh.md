# CPA Manager Plus v1.9.2

> 15 commits · 52 files changed · +1925 / -524

> [English ->](https://github.com/seakee/CPA-Manager-Plus/blob/v1.9.2/docs/release-notes/v1.9.2-en.md)

## Overview

本次发布聚焦 AI Providers 配置体验与 Codex quota 可观测性:新增 reset credits 展示、Codex 连通性测试和跨 provider 的 disableCooling 配置开关,同时优化模型发现去重、插件 OAuth provider 识别和监控过滤器结构。

## Highlights

### Features

- 在 Codex quota 状态中展示可用的 rate limit reset credits,并在专用接口不可用时保留 usage payload 计数回退(`web/quota`)。
- 为 Claude、Codex、Gemini 和 OpenAI provider 增加 disableCooling 编辑字段与详情页快捷开关,支持乐观更新和失败回滚(`web/ai-providers`)。
- 在 Codex provider 编辑抽屉中新增连通性测试,可选择测试模型并通过 `/v1/responses` 验证当前配置(`web/ai-providers`)。
- 优化模型发现体验:隐藏已配置模型、防止重复添加、将已添加状态收敛为紧凑 inline badge,并补充可见项全选与清空操作(`web/ai-providers`)。
- 支持插件 manifest 中显式 `oauth_provider` 字段,避免插件 OAuth key 与内置 provider id 冲突(`web/oauth`)。

### Fixes

- 当 `/openai-compatibility` 缺少 `disableCooling` 时,从 `/config` 回填并按规范化 provider identity 合并(`web/providers`)。
- OpenAI key 测试失败现在通过共享状态图标展示逐 key 错误提示,提示宽度随内容自适应(`web/ai-providers`)。

### Refactor

- 移除实时监控中无入口的 header error kind、header error code 和 quota plan 过滤器,过滤栏从 9 个下调到 6 个,保留底层事件诊断字段(`web/monitoring`)。

### Tests

- 补充 OpenAI key 状态提示、provider `disableCooling` 回填、Codex reset credits、provider 请求、插件 OAuth provider 解析和监控过滤器精简相关回归测试(`web`)。

## Upgrade Notes

- 无需数据库迁移或后端配置变更。
- 如果自动化或书签仍带有旧的监控 URL 参数 `headerErrorKind`、`headerErrorCode` 或 `quotaPlan`,这些参数将不再影响过滤结果。
- 建议为本版本补充截图:AI Providers 的 disableCooling 开关、Codex 连通性测试、模型发现已添加 badge、Codex reset credits quota 展示。
