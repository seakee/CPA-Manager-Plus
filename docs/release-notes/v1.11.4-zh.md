# CPA Manager Plus v1.11.4

## Overview

本次发布聚焦配置更新的可靠性与用量识别的准确性。OAuth 模型别名和 AI Provider 的连续操作不再互相覆盖；Codex 用量会按实际请求档位统计；监控界面也能更清楚地区分多 Key 渠道来源。

## Highlights

### Features

- 配额卡片新增可通过键盘操作、会根据视窗自动定位的说明浮层；在窄屏下，套餐、重置次数和到期信息展示更紧凑易读（`web/quota`）。

### Fixes

- OAuth 模型别名更新改为串行处理，并与 CPA 的校验规则保持一致；多渠道修改会在失败或不确定结果时尽力回滚，避免连续编辑丢失已保存配置（`web/auth-files`）。
- AI Provider 的启停、优先级和健康检查更新使用共享锁协调，防止重叠的乐观写入覆盖较新的 Provider 配置（`web/ai-providers`）。
- Codex 请求与响应档位不一致时，Usage Analytics 和 Manager Server 会按请求档位聚合用量；其他 Provider 继续优先采用响应档位（`manager-server/usage`、`web/usage`）。
- Request Monitoring 与 Account Overview 会优先显示可区分的多 Key 来源标识，并正确处理 Codex、xAI 的 OAuth 邮箱身份，避免渠道被合并为笼统标签（`web/monitoring`）。

### CI

- 发布工作流会在原生包和 Docker 镜像任务都成功后，发送仓库内已审核的 Telegram 发布内容；缺少配置或投递失败不会回滚已创建的 Release（`ci/release`）。
- PR 检查会按变更路径选择相关任务，并覆盖工作流、原生打包脚本和 Docker Compose 配置，减少基础设施变更漏检（`ci/pr-check`）。

## Upgrade Notes

- 本版本不包含 SQLite schema、持久化格式、`data.key` 或必需配置迁移。
- 对于请求档位与响应档位不一致的 Codex 新用量记录，聚合结果会使用请求档位；历史已存储记录不会自动重算。
