# 认证文件

认证文件页面用于管理账号凭证和账号状态。它回答的问题是：有哪些账号、账号是否启用、对应哪个 `auth_index`、最近是否有配额或认证异常。

添加新账号的登录流程在 [OAuth 登录](./oauth.md)。本页负责登录之后的日常维护。

## 先看哪些信息

- **文件名和提供商类型**：确认账号属于 Codex、Claude、Vertex、Antigravity、Kimi 或其他来源。
- **`auth_index`**：账号的稳定索引。用量、配额、巡检和账号处理都依赖它做关联。
- **启用状态**：手动禁用的账号不会被自动恢复覆盖。
- **备注、优先级和项目 ID**：用于区分账号用途和路由偏好。
- **配额和健康提示**：包括冷却、需要重新授权、额度窗口或最近 Header 观察结果。

多账号环境中，务必给账号稳定的 `auth_index`。没有稳定索引时，历史用量、配额、巡检结果和账号动作很难准确落到同一个账号上。

## 常用操作

- 刷新认证文件列表和账号状态。
- 粘贴 JSON 或上传认证文件。
- 下载、编辑、禁用、恢复或删除认证文件。
- 使用搜索、排序、页大小和视图模式快速定位账号。
- 按 Codex 状态、计划类型或“只看问题账号”筛选。
- 批量修改优先级、备注、项目 ID 或状态。
- 查看支持模型，判断某个账号是否应该承接目标模型。
- 打开前缀代理配置，复制客户端可用的代理地址。

不确定账号是否还要用时，优先禁用，不要直接删除。禁用保留历史关联；删除会影响后续巡检、配额和账号动作的可追踪性。

## 添加或更新认证文件

1. 如果是 OAuth 账号，先去 [OAuth 登录](./oauth.md) 完成授权。
2. 如果已经有 JSON，使用粘贴或上传保存到认证文件目录。
3. 保存后回到认证文件列表，确认文件名、提供商类型和 `auth_index`。
4. 发一条低成本请求。
5. 到 [请求监控](./monitoring.md) 确认请求使用了正确账号。

粘贴 JSON 时，选择与来源匹配的格式。不同来源的 JSON 字段可能不同，保存成功不代表一定能服务请求。

直接上传官方 sub2api 账号导出 JSON 时，CPAMP 会在浏览器中识别并转换其中的 OpenAI OAuth 账号，再按账号拆分为独立的 CPA Codex 认证文件逐个上传。粘贴 sub2api JSON 使用相同的转换流程；多账号导入会自动生成每个账号的文件名，不会把顶层 JSON 数组作为一个认证文件保存。

## 问题账号怎么处理

- **需要重新授权**：进入 [OAuth 登录](./oauth.md)，完成后回到本页确认状态。
- **额度耗尽或冷却中**：进入 [配额管理](./quota.md) 看恢复时间和冷却来源。
- **Codex 状态异常**：进入 [Codex 账号巡检](./codex-inspection.md) 查看建议动作。
- **请求失败但账号看起来正常**：进入 [请求监控](./monitoring.md) 查看失败摘要和实际模型。
- **账号处理队列有候选项**：进入 [账号处理队列](./account-actions.md) 决定忽略、标记已处理、启用或删除。

## 安全建议

认证文件包含敏感凭证。不要把完整 JSON、OAuth Token、API Key 或管理密钥贴到问题反馈里。需要排障时，优先提供脱敏后的请求监控摘要、账号状态截图和日志时间点。
